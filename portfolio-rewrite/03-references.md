# Phase 4 — Structural references (not mimicry)

**Purpose:** What to steal from strong GitHub profiles; what to avoid. Voice target: senior PM / builder at API-first companies (Anthropic-adjacent register: clear, honest, dense).

---

## 1. simonw / Datasette / llm

| Lens | Takeaway |
|------|----------|
| **Opening** | Name + one line of **what you build** + link to current obsession. No skill grid. |
| **Project framing** | “What this is / isn’t” — sets expectations immediately. |
| **Voice** | First person, conversational but **precise**; admits limitations. |
| **Visuals** | Screenshots when they teach; no vanity stats. |
| **Steal** | Running commentary tone; TIL-style honesty; README as index to deeper docs. |
| **Don’t copy** | Volume of micro-projects unless you maintain them — you have one flagship (RexAlgo). |

---

## 2. thesephist (Linus Lee)

| Lens | Takeaway |
|------|----------|
| **Opening** | Short hook — identity + current thread. |
| **Projects** | Each item: **why it exists**, not stack list. |
| **Voice** | Reflective, minimal adjectives. |
| **Visuals** | Almost none — confidence through restraint. |
| **Steal** | Sequencing: 3–5 works, best first. |
| **Don’t copy** | Pure research-lab framing — you are product + ship. |

---

## 3. karpathy / nanoGPT

| Lens | Takeaway |
|------|----------|
| **Opening** | Credentials implied by work, not claimed. |
| **Projects** | Minimal README; code is the argument. |
| **Voice** | Technical, zero marketing. |
| **Steal** | Brevity; no decoration. |
| **Don’t copy** | Researcher-only identity — you need PM legibility in first 10 seconds. |

---

## 4. swyx — learning in public

| Lens | Takeaway |
|------|----------|
| **Opening** | Career arc as **asset** (non-linear path). |
| **Voice** | Narrative + lessons; links to writing. |
| **Steal** | BD → CS → PMM → PM arc stated in **one line** without apology. |
| **Don’t copy** | “Creator economy” tone — stay fintech/platform serious. |

---

## 5. Anthropic / OpenAI cookbooks

| Lens | Takeaway |
|------|----------|
| **Register** | “Here’s what works / here’s what doesn’t”; runnable examples; env vars explicit. |
| **Steal** | TOC, clear prerequisites, failure modes documented. |
| **Don’t copy** | Lab voice — you are vendor PM, not research lab. |

## Synthesized template (your profile)

| Section | Depth | Rules |
|---------|-------|-------|
| 1. Name + role + positioning | 3 lines max | No “passionate”; no “full-stack” without qualifier |
| 2. Currently | 2–3 lines, dated | Real quarter |
| 3. Selected work | 3–6 rows | Problem → who → link → **why it matters** half-line |
| 4. Product philosophy | 3–4 short ¶ | Concrete opinions only |
| 5. Background | 1 line | Arc |
| 6. Contact + writing | Links | Medium > LinkedIn fluff |
| Optional | 1 mermaid | Platform map only |
| Forbidden | komarev, GIFs, skill bars, emoji walls, language charts | |

**Link strategy:** Profile README → 5 pins aligned with table order → `portfolio-rewrite/` for depth when asked.
